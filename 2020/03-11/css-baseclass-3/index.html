[基礎課程] CSS教學(三)：盒子模型與定位點

發表於 2020-03-11 分類於 職訓教材

若順著教材讀到這裡，你會發現很多CSS的控制成敗都跟block或inline有關。這裡我們會完整介紹什麼是盒子模型(Block & Inline)，有了完整的盒子概念後，就能接著會介紹如何去定位這些元素到指定的位置，學完這篇能幫助你整個CSS切版功力大幅提升。

盒子模型 Box Model

  • CSS 是盒子的世界，分為兩種基本盒子分為區塊盒（block box），行內盒（inline box）。
  • 在編寫 CSS 時要先確認使用的 HTML 標籤的原型是區塊標籤還是行內標籤，這是寫 CSS 十分非常重要的事。
  • HTML 的每個標籤都可以視為盒子，當使用了標籤等於建立一個盒子，等同產生了容器區塊，也稱為內容區域。
  • 外距、內距的單位百分比 %，是根據父元素的寬度做為計算間距寬度，也就是單位%不論是上下左右的內外距都是依外層寬度而非高度。
  • img 圖片雖為行內元素，但表現像 inline-block，非常適合解釋 background-color 包含 padding。
  • 有些標籤會有瀏覽器預設的上下外距，例如 h、p、body、ul。

以 div區塊標籤、span行內標籤為經典代表。

區塊盒（block box）

常見的區塊型標籤：div、h1、p、ul、li…

  1. block可用 margin 與 padding 屬性，標籤前後都有結束斷行處理，且會填滿撐大父元素，預設自動占滿整體寬度。
  2. 會將高度延展到可容納它本身所存放的所有東西，也就是利用內容去撐開它的高度。
  3. block並不會受到 vertical-align 影響。

區塊盒的內容區域（width）、內距（padding）、邊框（border）、外距（margin）屬性都是有效可指定的。

行內盒（inline box）

常見的行內型標籤：span、a、em…

  1. 循規蹈矩且前後不會結束斷行，內容多少寬就多少，即是設定 width、height 都不會有作用。
  2. 上下的 margin 無效，但左右的 margin 依舊有效。
  3. padding、border 上下左右皆有效。
  4. height 與 width 屬性無效，原因在於行內無法設定寬、高。
  5. 可以被 vertical-align 影響。

行內區塊盒（inline-block box）

算是兩者的合併體系，主要可以當作不會斷行(inline)且可以彈性指定內外距(block)。

  1. 繼承 inline，預設內容多少寬度就多少，不會強迫換行。
  2. 繼承 block，可使用 margin, padding, width, hegiht, border都可指定。

舉例示範

使用標籤產生盒子，即產生容器區塊，目前內容區域沒有東西，瀏覽畫面還看不到東西

block一旦有了內容後，產生了有內容的內容區域，內容區域會以區塊（block）或行內（inline）來顯示。

  1. 區塊是橫行霸道的佔位王，即便它的內容很少，也要佔據整列。
  2. 區塊可以自由使用寬（width）、高（height）、內距（padding）、外距（margin）

inline行內是循規蹈矩的乖乖排，它的內容就已經是它的寬度，許多行內盒子會依序乖乖排好自己的位置。 行內不能設定寬、高，上、下外距無效。

min-width

當瀏覽器視窗很窄時，方框最多只能縮小到設定的尺寸，內容沒有裝滿情況下，因為設定最小高度，以此寬度呈現。

max-width

當瀏覽器視窗很寬時，方框最多只能放大到設定的尺寸，此元素可以比該設定窄，但無法比該設定寬，可以限制版面的最大寬度，使版面內容不要超過 max-width。

!> max-height 內容超過該高度，內容會溢出。

最大值最小值屬性的一大好處在於能夠相對安全的混合使用各種長度單位，可以在以長度設定限制的同時混合使用百分比值， 也可以再用百分比值設定限制時混用絕對長度。

尤其是搭配浮動(float)元素時十分有用，例如，能夠允許float元素的寬度是親代元素(也就是容器區塊)的特定比例，同時保證float元素寬不會小於 10em，也能夠做到相反的方向。

舉例1: 最大寬度為 500px，若小於500px寬度會以 40% 設定

舉例2: 先控制最大寬度！不管介面有多大都不會超出這個範圍。當瀏覽器小於指定的 max-width 1200px 寬度時，會用原先設定的寬度百分比 80% 來計算。

在border-box模式下，border、padding 都會算在寬度裡面，只需考量處理margin。bootstrap 便是採用 border-box模式。

overflow

如果對區塊設定高度 Height，內容如果超過高度，則會產生內容溢出，所以設定頁面元素高度的通則是不要設定，若超出時可以設定 overflow如何處理溢出。

Tips: 使用 max-height 來建立純 CSS 的捲動軸

visibility

visibility 屬性可以讓盒子隱藏不見，但將原有的位置空下來，display:none 並不會空下來。

定位 Position

定位原則

  1. 定位分為絕對定位、相對定位，這兩者的差異只是定位點的不同而產生的差異。只要知道定位點的設定就可以使用絕對定位或相對定位，要先確認定位點。
  2. 容器區塊(containing block)是包含另一個元素的區塊，根元素 HTML 是 body 的容器區塊，body 又是其它所有子元素的容器區塊。
  3. 對於定位而言，容器區塊（父）完全取決於定位的類型，子區塊（子）根據上一層是否有定位，若沒有就會依照初始區塊定位。

對於 position 值為 absolute 的非根元素，其容器區塊是由最接近，且 position 值不是 static 的祖先形成，也就是要 absolute、relative。

  • 若祖先是區塊盒，容器區塊會設為該元素內距（padding）的邊緣。
  • 若祖先是行內盒，容器區塊會設為祖先的內容邊緣。
  • 若沒有祖先，容器區塊定義為初始容器區塊。
  • 設定 position 才可以使用 z-index 屬性。（重要）

對於 position 值為 relative 或 static 的非根元素，其容器區塊是由最接近的區塊盒、表格欄位或行內區塊祖先形成。

絕對定位 position:absolute

當子元素使用絕對定位（absolute）時，子元素會離開文件流向自己獨立起來， 若父容器又沒有任何定位，它會完全脫離父子關係離家出走， 子元素會去外面找預設好的定位點，也就是版面基準點來定位。一個 absolute 就會有自己一層的平面空間，有多少個 absolute 就會有多少層，意味著有多少個 z-index。

相對定位 position:relative

設定 position:relative，區塊會以原本顯示的位置為基準，接著可以指定上下左右偏移。 與 absolute 不同的是，區塊原本的空間仍會保留，若只設定了 position:relative，卻沒有指定偏移座標， 則顯示結果不會有甚麼改變。

相對定位說明

絕對定位說明

  1. 請先設計版面，其目的希望對.bigbox進行畫面中心處定位。

做法1: 對div.bigbox直接指定absolute，跳脫父盒子body的影響，調整參數是window為比例。

做法2: 對父親body下手，先填滿畫面且指定display為flex。宣告子內容是垂直且水平皆是至中。

!> 重要：relative 可和 float 一起使用；absolute 不可和 float 使用。

固定定位 position:fixed

absolute 與 fixed 的行為很像，不一樣的地方在於 absolute 元素的定位是在他所處上層容器的相對位置。如果這個套用 position: absolute 的元素，其上層容器並沒有「可以被定位」的元素的話，那麼這個元素的定位就是相對於該網頁所有內容（也就是 元素）最左上角的絕對位置

黏性定位 position:sticky

這是一個結合了 position:relative 和 position:fixed 的定位，適用些特殊場景(像是header nav效果)，會先以relative效果方式，當relative會使元素離開viewport時，瞬間會轉成fixed效果。部分情況下使用sticky會失敗(無法算出fixed)

  • 未指定特定閾值(top,bottom,left,right)
  • 父元素的overflow屬性值為：hidden, auto, scroll
  • body height:100%

設定順序為:父層 position=>子層 position=>子層 offset(top、bottom、left、right);
